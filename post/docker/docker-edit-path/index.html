    Docker 기본 경로 변경

    • 24/7/2022
    • 읽는 시간 1분

    docker 용량 문제

    docker의 overlay로 인해 물리디스크가 100% 차지하는 경우가 발생한다.
    안쓰는 이미지와 컨테이너 제거 및 도커의 기본 경로를 바꿔야 한다.

    1. 안쓰는 이미지와 컨테이너 삭제

    1$ docker system prune -a -f
    • 컨테이너가 실행되고 있지 않은 것까지 모두 삭제
    • 도커 기본 설치 경로는 /var/lib/docker
      → 볼륨의 기본 설치 경로는 /var/lib/docker/volumes

    2. 도커의 기본 경로 바꾸기

    1# docker 종료
2$ systemctl stop docker
3
4# 설정파일 수정
5$ sudo vi /lib/systemd/system/docker.service

    docker.service 설정 파일

    1[Service]
2# ExecStart=/usr/bin/dockerd
3ExecStart=/usr/bin/dockerd --graph=[변경할 경로]

    설정파일 변경 후

    1# 데몬 리로드
2$ systemctl daemon-reload
3
4# 도커 시작
5$ systemctl start docker
